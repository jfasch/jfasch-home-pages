Load From BackingStore in Constructor¬∂

Problem

UserDB Naming and Implementation¬∂

  • UserDB is an in-memory structure - a cache basically

  • It has read() and write() methods bolted onto it, with unclear signature (bool binary?)

  • Create a BackingStore class that UserDB gets passed in the constructor (where it then loads from). It does the read()/write() when the user calls sync() on UserDB.
