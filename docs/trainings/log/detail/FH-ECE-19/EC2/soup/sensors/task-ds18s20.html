Temperature Sensor: DS18S20¶

  • Description

    • Complication: OneWire on Breadboard

  • Integration

  • Further Information

  • Dependencies

Description¶

  • Write a Python class that implements the Thermometer interface, and sits on top of the DS18S20 Onewire temperature sensor.

  • Do all the necessary integration stuff (see Linux and OneWire (using DS18B20 Temperature Sensor as Slave))

  • Document all that

Complication: OneWire on Breadboard¶

Start with the bitbanging OneWire driver, w1-gpio, as described in Linux and OneWire (using DS18B20 Temperature Sensor as Slave).

That method (OneWire master in software) is susceptible to signal noise and/or timing issues, so we have a second method at hand: attach a DS2482 OneWire master to the Raspberry’s I2C bus, and let that talk with our sensor. Again, described in Linux and OneWire (using DS18B20 Temperature Sensor as Slave).

Integration¶

  • Once your thermometer is in a shape to be used, notify your users (Program: Read Sensors, Write to stdout depends on you) how this is done.

  • Help them integrate your thermometer

  • Before, push your documentation department to do their job, or else you will have to explain all that repeatedly
