Enter Algorithms (On Good Old C Arrays)¶

Live Hacking Screenplay

  • Screenplay: Pointer Arithmetic, And Algorithms

Iteration over all elements of an array …

int sum(const int *begin, const int *end)
{
    int sum = 0;

    while (begin < end)
        sum += *begin++; /* precedence? what? */
    return sum;
}

void copy(const int *src_begin, const int *src_end, int *dst_begin)
{
    while (src_begin != src_end)
        *dst_begin++ = *src_begin++;
}

Pretty, isn’t it?

Dynamically growing array: std::vector¶

See also

  • Brace Initialization (Uniform Initialization)

Documentation

  • std::vector

  • C arrays are fixed-size

  • Once allocated, they cannot change size

  • ⟶ rather unflexible

  • std::vector<> to the rescue

  • Same as Screenplay: Pointer Arithmetic, And Algorithms, only with a std::vector<> instead of a gool old C array:

    code/vector-basic.cpp¶
    #include <vector>
#include <iostream>
   
int main()
{
    std::vector<int> a;                // <--- C++98: initialization -> empty
    a.push_back(100);                  // <--- elements come during runtime
    a.push_back(200);                  // <--- ...
    a.push_back(300);                  // <--- ...
        
    for (size_t i=0; i<a.size(); ++i)  // <--- index based iteration
        std::cout << a[i] << std::endl;

    return 0;
}

  • Using C++11 brace initialization

    code/vector-basic-brace-init.cpp¶
    #include <vector>
#include <iostream>
   
int main()
{
    std::vector<int> a = {100, 200, 300};  // <--- C++11 brace initialization
        
    for (size_t i=0; i<a.size(); ++i)      // <--- index based iteration
        std::cout << a[i] << std::endl;

    return 0;
}

STL Iterators: Pointer Arithmetic On A std::vector<>¶

See also

  • Range Based for Loops: Introduction

  • Remember the beauty?

  • Using raw C pointers

    code/vector-basic-pointer-arith.cpp¶
    #include <vector>
#include <iostream>
   
int main()
{
    std::vector<int> a = {100, 200, 300};

    const int *begin = &a[0];              // <--- pointer to first element
    const int *end = &a[3];                // <--- pointer past last element

    while (begin != end)
        std::cout << *begin++ << std::endl;

    return 0;
}

  • Generalized pointer: STL iterator

  • .begin(), .end()

    code/vector-basic-iterators.cpp¶
    #include <vector>
#include <iostream>
   
int main()
{
    std::vector<int> a = {100, 200, 300};

    std::vector<int>::const_iterator begin = a.begin();  // <--- looks like a pointer, but isn't
    std::vector<int>::const_iterator end = a.end();      // <--- looks like a pointer, but isn't

    while (begin != end)
        std::cout << *begin++ << std::endl;

    return 0;
}

  • Simpler alternative, but not always applicable: range based for

    code/vector-basic-range-based.cpp¶
    #include <vector>
#include <iostream>
   
int main()
{
    std::vector<int> a = {100, 200, 300};
    for (int i: a)                           // <--- range based for
        std::cout << i << std::endl;
    return 0;
}

Algorithms And Containers : std::copy¶

Documentation

  • std::copy

  • std::vector<> is a generalized array

  • Why not copy contents of one into the other?

  • First: std::vector<> into C array of equal size

    code/copy-vector-to-array.cpp¶
    #include <vector>
#include <iostream>
   
int main()
{
    std::vector<int> src = {100, 200, 300};
    int dst[3];

    std::copy(src.begin(), src.end(), dst);
    for (int i: dst)
        std::cout << i << std::endl;

    return 0;
}

std::copy Into std::vector<>: Preallocate¶

  • The code above allocates a C array on the stack, of sufficient size

  • ⟶ this is crucial!

  • Remember, std::vector<> is empty by default

  • This code will crash:

    code/copy-vector-to-vector-crash.cpp¶
    #include <vector>
#include <iostream>
   
int main()
{
    std::vector<int> src = {100, 200, 300};
    std::vector<int> dst;                             // <--- ATTENTION: empty!!

    std::copy(src.begin(), src.end(), dst.begin());   // <--- CRASH!
    for (int i: dst)
        std::cout << i << std::endl;

    return 0;
}

  • Solution 1: Pre-allocate destination (using .size())

    code/copy-vector-to-vector-preallocate.cpp¶
    #include <vector>
#include <iostream>
   
int main()
{
    std::vector<int> src = {100, 200, 300};
    std::vector<int> dst(src.size());                 // <-- preallocate just enough memory

    std::copy(src.begin(), src.end(), dst.begin());
    for (int i: dst)
        std::cout << i << std::endl;

    return 0;
}

std::copy Into std::vector<>: Using std::back_insert_iterator¶

Documentation

  • std::back_insert_iterator

  • Solution 2: std::back_insert_iterator as destination

  • Calls .push_back() when written to

code/copy-vector-to-vector-back-insert-iterator.cpp¶
#include <vector>
#include <iostream>
   
int main()
{
    std::vector<int> src = {100, 200, 300};
    std::vector<int> dst;                             // <-- empty

    std::copy(src.begin(), src.end(),
              std::back_insert_iterator<std::vector<int>>(dst));

    for (int i: dst)
        std::cout << i << std::endl;

    return 0;
}

#include <vector>
#include <iterator>
#include <iostream>
   
int main()
{
    std::vector<int> src = {100, 200, 300};

}
